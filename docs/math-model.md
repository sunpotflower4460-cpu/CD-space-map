# 数学モデル

CD星図 / Harmonic Disk では、基準周波数 `f0` に対して表示したい周波数 `f` がどこに来るかを、オクターブ差・層・角度で表します。

## 1) オクターブ差

`h = log2(f / f0)`

- `f0`: 基準周波数
- `f`: 表示したい周波数
- `h`: 基準から何オクターブ離れているか

`h = 0` なら基準と同じ高さ、`h = 1` なら1オクターブ上、`h = 2` なら2オクターブ上です。

## 2) 層（ディスクレイヤー）

`layer = floor(h)`

- `layer`: どのディスク層か

`floor` は小数点以下を切り捨てるため、`h=1.7` なら `layer=1` になります。

## 3) 初期配置角度

`angle = frac(h) × 2π`

- `frac(h)`: `h` の小数部分（`h - floor(h)`）
- `angle`: ディスク上の静的な初期角度（ラジアン）

これが「**1周 = 1オクターブ**」の意味です。1オクターブ進むごとに角度はちょうど 2π（1周）して元の向きに戻ります。

### 角度の向き

座標系は Three.js の右手座標系を使用しています。

```
angle=0      → x 軸正方向 (右)
angle=π/2    → z 軸正方向 (手前)
angle=π      → x 軸負方向 (左)
angle=3π/2   → z 軸負方向 (奥)
```

上から見ると **反時計回り** に角度が増加します（標準的な数学的正方向）。

## 4) 観測用回転アニメーション

静的な初期角度とは別に、再生中は各点がリアルタイムで回転します。

`rotationSpeed = frequency / displayScale × 2π`

`currentAngle = initialAngle + rotationSpeed × time × playbackSpeed`

- `displayScale`: 人間が見える速度にするための表示倍率（観測用の減速倍率）

実際の Hz でそのまま回すと速すぎるため、`displayScale` で割って観測できる速度に落とします。周波数比は保たれるため、比率の違いは回転速度の違いとして観測できます。

> **注意**: `1周 = 1オクターブ` は **初期配置** の話です。アニメーション回転では周波数に比例した速度で回るため、時間が経つと初期角度からずれていきます。`displayScale` を変えると全体の回転速度が変わりますが、周波数比（各点の相対的な速度比）は保たれます。

## 5) 精度対策

`rotationSpeed × time` が非常に大きくなると浮動小数点精度が劣化する可能性があります。実装では回転角を `mod 2π` して数値を安定させています。

```ts
currentAngle = (initialAngle + rotationSpeed * time * playbackSpeed) % (2π)
```

## 6) 実験保存の仕様

「この条件を保存」は **観測条件**（基準周波数・プリセット・表示速度など）の保存です。軌跡（過去の位置記録）は保存されません。読み込み後は再生することで新しい軌跡を観測できます。
